Prozessablauf für unseren Rovo Agent "PitStop Inspect Agent" für unsere Codegeist 2025 Projekt: 1. Der User startet in Jira. Dort ruft er in der Sidebar über “Ask Rovo” unseren PitStop Inspect Agent auf. 2. Unser Agent begrüßt ihn im Chat und fragt nach den zwei Bildern des Wagens. Eins, dass den Zustand vor dem Rennen zeigt. Das zweite für den Zustand nach dem Rennen. 3. Der User lädt die angefragten 2 Bilder im Agent Chat hoch. 4. Unser Agent fragt auch noch nach einer Confluence Page ID, um im zugehörigen Confluence nach den Fahrzeugdaten zu suchen (z.B. erforderliche Reifenprofiltiefe), um diese Daten mit dem Bild zu vergleichen. 5. Der User gibt ihm die Page Id. 4. Unser Agent beginnt sofort mit der Damage Detection. Er prüft sorgfältig, ob neue Schäden hinzukamen und geht die aus Confluence ermittelten Fahrzeugdaten durch und prüft diese mit dem Wagen im Bild ab. 5. Unser Agent erkennt, dass etwas am Auto kaputt ist und gibt eine List der Schäden und eine Zusammenfassung in natürlicher Sprache aus. 6. Unser Agent fragt den User, ob er ein Jira Ticket erstellen soll, um den Fehler beim Reparatur-Team zu melden. Und bittet den User um einen Jira-Projekt Schlüssel. 7. Der User gibt den Jira Projekt-Schlüssel an, in dem unser Agent das Ticket erstellen wird. 8. Unser Agent erstellt anschließend ein Hauptticket und darin Unter-Tickets für jeden gefundenen Schaden. 9. Zum Schluss bestätigt unser Agent, dass das Ticket im angegebenen Projekt erstellt wurde und sich das Reparaturteam bei Rückfragen beim User melden wird. 10. Da unser Agent auch einen Link zu dem erstellten Ticket ausgibt, können wir es aufrufen und sehen dort die Subtasks für die einzelnen Schäden die man reparieren muss.
